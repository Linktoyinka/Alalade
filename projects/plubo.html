The Brief

About Plubo

Plubo is an online boutique retail store. Users can find and purchase trendy fashion items all within the app. The app targets users who are fashion enthusiasts.

Objective

Develop a dedicated e-commerce mobile application and a responsive website to facilitate seamless product purchases and provide users with fashion information.

Checkout the prototypes

Understanding the users

Research Summary

I conducted interviews and created empath maps to understand the users I’m designing for and their needs. The primary user group identified through the research was young and middle aged adults (18 - 45 years) who have frequent social engagements and need to update their wardrobe regularly.

User Persona

I created a user persona which is representative of Plubo’s primary user group based on data from the user research.

Pain Points

The insights we have gathered can be summarized into three main pain points.

Time
Keeping up with the latest fashion trends takes a lot of time.
Cataloguing
Platforms for purchasing fashion items often create problems for users by having poor catalogues and product sorting.
Copy
Product descriptions are usually text heavy and vague or too short.

Ideation

Storyboarding

Creating storyboards helped me visualize how the user will interact with the app, which in turn ensured that I came up with ideas that solves the user problems.

User Flow

I created a diagram outlining the user flow to gain a comprehensive understanding of the user's journey within the application.

Wireframes

Once the user flow was established, I designed wireframes and created a low fidelity prototype to test it out before committing to creating the high-fidelity mockups.
